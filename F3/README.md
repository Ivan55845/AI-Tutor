# Фаза 3: Оценка размера проекта AI Tutor

Проект: **AI Tutor — персональный AI-репетитор для обучения с адаптивной системой подачи материала**

---

## 1. Акторы и границы системы

### Акторы

1. **Ученик (Student)**  
   Основной пользователь приложения — школьник или студент, который проходит обучение,  
   получает персонализированные задания и отслеживает свой прогресс.

2. **Внешний сервис контента (Content API)**  
   Сторонняя система, предоставляющая образовательные материалы, тесты и справочники.

### Граница системы

**AI Tutor** включает:
- веб/мобильное приложение,
- backend-сервис,
- AI-модули (адаптивный алгоритм, генерация объяснений, анализ ошибок).

Пользователь взаимодействует через UI приложения, внешний сервис — через API.

### Оценка команды и сроков

При допущении:
- 1 SP ≈ 10–12 человеко-часов
- скорость команды: 25–30 SP / спринт

Получаем:
- **Общие трудозатраты:** ~1200–1500 ч
- **Сроки:** 4–5 спринтов (≈ 2–3 месяца)
- **Команда:**
  - 1 ML-инженер
  - 1 backend-разработчик
  - 1 frontend/mobile-разработчик

## 7. Приоритизация и отбрасываемые фичи

### Наибольший вклад в сложность

| UC | Название | SP | Приоритет |
|---:|----------|:--:|:---------:|
| UC7 | Адаптивная сложность | 21 | Можно упростить |
| UC5 | Подсказки от AI | 13 | Ядро продукта |
| UC6 | Анализ ошибок | 13 | Можно отложить |
| UC11 | Интеграция с API | 13 | Можно упростить |
| UC12 | Offline-режим | 13 | Можно исключить |

### При необходимости ускорения разработки можно:

1. **Исключить UC12** (offline) — экономия ~13 SP
2. **Упростить UC7** (фиксированные уровни вместо AI) — экономия ~8 SP
3. **Отложить UC11** (ручной ввод контента) — экономия ~13 SP

**Потенциальная экономия: до 34 SP (~28% проекта)**

При этом основная ценность продукта (UC3–UC6, UC8) сохраняется.
